Getting started

As the name "Angular Model Pattern" says, it's a pattern not a library so there is literally zero dependencies needed to start using it in your own project.

1. Create model.service.ts

Create model.service.ts file in your core module with the following content...

        
{{fileModelService}}

2. Provide model service in your core.module.ts

Import MODEL_PROVIDER constant and add it to providers of your CoreModule. (Service can also be provided in AppModule in case you don't use core module)

        
{{fileCoreModule}}

3. Use model in your business service

Import ModelFactory in your service, inject it into constructor using Angular dependency injection mechanism and create model instance.

Expose model's data as a variable with descriptive name (eg: todos$: Todo[] = this.model.data$) to be consumed by the components which import the service.

Implement model mutation inside of self-contained service methods which retrieve model state with this.model.getData(), perform necessary mutations and set new state using this.model.setData(newData).

Check full example of todos.service.ts and todos.component.ts on Gihub.

        
{{fileBusinessService}}

4. Optionally add model service tests

Create model.service.spec.ts file next to your model.service.ts file and populate it with following content. Tests can help you in situation when you want to customize model service to better accommodate to your specific needs. Check original file on Github.
