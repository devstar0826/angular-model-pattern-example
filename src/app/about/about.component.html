Angular Model Pattern


Imagine Redux with explicit function calls instead of action creators, one model (store) per feature and immutable data exposed as RxJS observables

Check on Github

Features

content_copy Immutable model

Newer worry about undesired side-effects! Your data is exposed as an Observable which clones underlying data by default. This prevent accidental mutations by model consumers.

call_made Explicit function calls

Explicit function calls are used to perform all model state mutations. This enables you to step through the execution call stack with ease.

refresh Model per domain / feature

Create multiple models to separate your state into manageable chunks. Combine state from multiple models using RxJS .combineLatest() or some of the many other available operators.

refresh One way data flow

Manage your state in simple manner with RxJS Observables powered one way data flow. Components simply subscribe to model updates with | async pipe or explicitly in ngOnInit()

add_box Angular way

Create multiple instances of your model by using standard Angular dependency injection (DI) mechanism with build in factory support.
