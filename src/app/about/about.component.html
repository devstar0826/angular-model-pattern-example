Angular Model Pattern

TLDR
Imagine Redux with explicit function calls instead of actions, one store (model) per feature and immutable data exposed as observable
Features
  • One way data flow (flux)
  • Explicit service function calls (instead of actions / events)
  • Immutable data model (store)
  • Angular way (standard injectable angular service)
  • Model per domain concern (eg: every service has it's own model)
call_made Explicit function calls

Explicit function calls are used to perform all model state mutations. This enables you to step through the execution call stack with ease...
