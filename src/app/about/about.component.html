Angular Model Pattern


Imagine simple and predictable state management with explicit function calls, one model (store) per feature and immutable data exposed as RxJS Observables

Getting started Check on Github

Features

refresh One way data flow

One way data flow proved to be the most effective and understandable way of managing application state. Components simply subscribe to model state exposed as RxJS Observable with | async pipe in their template or explicitly by calling .subscribe() in ngOnInit().

content_copy Immutable model

Stop worrying about unintended side-effects! Your data is accessible through an RxJS Observable which clones underlying data by default and prevents accidental mutations by model consumers. Immutability can be overridden by passing optional flag or custom clone function.

folder One model per feature

Create multiple models to separate your state into manageable chunks. Combine state from multiple models using RxJS .combineLatest() or one of the many available operators.

call_made Explicit function calls

Model is updated by explicit calls to vanilla service methods which are easy to understand and debug. Step through the call stack from component event handler to model data setter - one function call at the time!

add_box Angular way

Create multiple instances of your model by using standard Angular dependency injection mechanism and build in factory support.
